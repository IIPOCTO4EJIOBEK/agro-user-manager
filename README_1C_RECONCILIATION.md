# 📘 ВНЕШНЯЯ ОБРАБОТКА 1С: СВЕРКА ЗЕМЕЛЬНЫХ УЧАСТКОВ С ЕГРН

## 📋 Назначение
Внешняя обработка для автоматической сверки данных из выписок ЕГРН (Excel-файл от Python-скрипта) с регистром сведений «Регистрация земельных участков» в 1С.

---

## 🔧 ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Версия 1С
- **1С:Предприятие 8.3** (версия 8.3.10 и выше)
- Режим совместимости: не ниже 8.3.10

### Необходимые объекты в конфигурации 1С
Обработка предполагает наличие следующего регистра сведений:

```bsl
РегистрСведений.РегистрацияЗемельныхУчастков
```

**Структура регистра должна включать измерения и ресурсы:**

| Поле | Тип | Описание |
|------|-----|----------|
| **КадастровыйНомер** (Измерение) | Строка | Кадастровый номер участка |
| **Организация** (Измерение) | Справочник.Организации | Владелец участка |
| **Период** (Измерение) | Дата | Дата регистрации записи |
| **КодПоОКТМО** (Ресурс) | Строка | Код ОКТМО |
| **КадастроваяСтоимость** (Ресурс) | Число(15,2) | Кадастровая стоимость |
| **КодКатегорииЗемель** (Ресурс) | Строка | Код категории земель |
| **ДоляВПравеЧислитель** (Ресурс) | Число(10,0) | Доля (числитель) |
| **ДоляВПравеЗнаменатель** (Ресурс) | Число(10,0) | Доля (знаменатель) |
| **НалоговаяСтавка** (Ресурс) | Число(5,4) | Налоговая ставка |
| **НаименованиеИлиАдрес** (Ресурс) | Строка | Наименование или адрес участка |

> ⚠️ **ВАЖНО:** Если имена полей в вашей конфигурации отличаются, необходимо отредактировать текст запроса в функции `НайтиУчастокВРегистре()`.

## 📥 УСТАНОВКА ОБРАБОТКИ

### Способ 1: Через Конфигуратор (рекомендуется)

1. **Запустите 1С в режиме «Конфигуратор»**
   - Завершите работу всех пользователей
   - Запустите 1С с ключом `/C` или выберите режим «Конфигуратор»

2. **Создайте новую внешнюю обработку**
   ```
   Меню: Файл → Новый → Внешняя обработка
   ```

3. **Скопируйте код**
   - Откройте файл `1c_land_reconciliation.epf.txt`
   - Скопируйте ВСЁ содержимое файла
   - Вставьте в окно редактора кода новой обработки

4. **Сохраните обработку**
   ```
   Меню: Файл → Сохранить как...
   Имя файла: СверкаЗемельныхУчастковЕГРН.epf
   Тип файла: Внешняя обработка (*.epf)
   ```

5. **Проверьте синтаксис**
   ```
   Меню: Отладка → Проверить синтаксис модуля
   ```
   Убедитесь, что ошибок нет.

6. **Закройте Конфигуратор**

### Способ 2: Прямая загрузка готовой обработки

Если у вас есть скомпилированный файл `.epf`:

## 🚀 ЗАПУСК И ИСПОЛЬЗОВАНИЕ

### Шаг 1: Открытие обработки

1. **Запустите 1С в режиме «Предприятие»**
2. **Откройте внешнюю обработку:**
   ```
   Меню: Файл → Открыть → Выберите файл СверкаЗемельныхУчастковЕГРН.epf
   ```
   
   ИЛИ через панель внешних обработок (если настроена):
   ```
   Раздел: Администрирование → Дополнительные отчеты и обработки
   ```

### Шаг 2: Загрузка данных из Excel

После открытия обработки:

1. Нажмите кнопку **«Загрузить Excel»** (или аналогичную на форме)
2. Укажите путь к файлу, созданному Python-скриптом:
   ```
   Пример: C:\Выписки_ЕГРН\egrn_parsed_data.xlsx
   ```
3. Дождитесь сообщения о количестве загруженных записей

**Формат ожидаемого Excel-файла:**

| Кадастровый номер | ОКТМО | Категория земель | Код категории | Стоимость | Доля | Дата |
|------------------|-------|------------------|---------------|-----------|------|------|
| 50:23:0010101:123 | 46648105 | Земли сельхозназначения | 003001000000 | 1500000 | 1/2 | 15.01.2024 |

> ✅ Обработка автоматически распознает колонки как на русском, так и на английском языке.

### Шаг 3: Выбор организации

Перед запуском сверки необходимо указать организацию:

1. В поле **«Организация»** выберите нужную организацию из справочника
2. Или организация будет подставлена автоматически (из текущей сессии)

### Шаг 4: Запуск сверки

1. Нажмите кнопку **«Выполнить сверку»**
2. Дождитесь завершения процесса
3. Просмотрите итоговое сообщение:
   ```
   Сверка завершена!
   Всего обработано: 150
   Найдено в 1С: 145
   Не найдено в 1С: 5
   Расхождений выявлено: 12
   ```

### Шаг 5: Анализ результатов

Результаты отображаются в таблице на форме обработки:

| Статус | Кадастровый номер | ... | Список расхождений |
|--------|------------------|-----|-------------------|
| OK | 50:23:0010101:123 | ... | |
| **ОШИБКА** | 50:23:0010102:456 | ... | ОКТМО; Стоимость; Налоговая ставка |
| **НЕ НАЙДЕН В 1С** | 50:23:0010103:789 | ... | Участок отсутствует в регистре 1С |

**Цветовая индикация:**
- 🟩 **Зеленый/без цвета** — всё совпадает (статус «OK»)
- 🟥 **Красный фон** — есть расхождения или участок не найден

### Шаг 6: Выгрузка отчета

1. Нажмите кнопку **«Выгрузить результаты»**
2. Укажите путь для сохранения Excel-файла
3. Откройте файл — строки с ошибками будут подсвечены красным

## 📊 ПРОВЕРЯЕМЫЕ ПАРАМЕТРЫ

Обработка выполняет сверку по **5 ключевым полям**:

| № | Параметр | Допустимое отклонение | Примечание |
|---|----------|----------------------|------------|
| 1 | **ОКТМО** | Точное совпадение | Код муниципального образования |
| 2 | **Код категории земель** | Точное совпадение | Например: 003001000000 |
| 3 | **Кадастровая стоимость** | ±0.01 руб. | Учитывается погрешность округления |
| 4 | **Дата регистрации** | ±1 день | Допускается разница в 1 день |
| 5 | **Налоговая ставка** | ±0.0001 | Автоматическая проверка: 0.3% для сельхоз, 1.5% иначе |

### 🎯 Логика проверки налоговой ставки

```bsl
ЕСЛИ КодКатегорииЗемель В (003001000000, 003002000000, 
                           003003000000, 003004000000) ТОГДА
    ОжидаемаяСтавка = 0.3000  // 0.3%
ИНАЧЕ
    ОжидаемаяСтавка = 1.5000  // 1.5%
КОНЕЦЕСЛИ
```

> 💡 Чтобы добавить другие коды категорий для ставки 0.3%, отредактируйте массив `МассивКодовСельхоз` в процедуре `ВыполнитьСверку()`.

## 🔧 НАСТРОЙКА ПОД ВАШУ КОНФИГУРАЦИЮ

### Изменение имен полей регистра

Если в вашей базе имена полей отличаются, откройте обработку в Конфигураторе и найдите функцию `НайтиУчастокВРегистре()`:

```bsl
// БЫЛО (наше предположение):
Запрос.Текст = 
    "ВЫБРАТЬ ПЕРВЫЕ 1
    |	РегистрацияЗемельныхУчастков.КодПоОКТМО,
    |	РегистрацияЗемельныхУчастков.КадастроваяСтоимость,
    |   ...";

// СТАЛО (ваш вариант):
Запрос.Текст = 
    "ВЫБРАТЬ ПЕРВЫЕ 1
    |	РегистрацияЗемельныхУчастков.ОКТМО,            // изменили имя
    |	РегистрацияЗемельныхУчастков.СтоимостьКадастровая, // изменили имя
    |   ...";
```

### Добавление новых кодов для ставки 0.3%

В процедуре `ВыполнитьСверку()` найдите и дополните массив:

```bsl
МассивКодовСельхоз = Новый Массив;
МассивКодовСельхоз.Добавить("003001000000");
МассивКодовСельхоз.Добавить("003002000000");
МассивКодовСельхоз.Добавить("003003000000");
МассивКодовСельхоз.Добавить("003004000000");
// ДОБАВЬТЕ СВОИ КОДЫ:
МассивКодовСельхоз.Добавить("003005000000"); // ваш код
МассивКодовСельхоз.Добавить("003006000000"); // ваш код
```

## ❗ ВОЗМОЖНЫЕ ОШИБКИ И РЕШЕНИЯ

| Ошибка | Причина | Решение |
|--------|---------|---------|
| «Регистр сведений не найден» | В конфигурации нет регистра с именем `РегистрацияЗемельныхУчастков` | Переименуйте регистр в запросе или создайте новый |
| «Поле ... не найдено» | Имя поля в запросе не совпадает с конфигурацией | Отредактируйте текст запроса в функции `НайтиУчастокВРегистре()` |
| «Ошибка при чтении Excel» | Файл поврежден или неверный формат | Проверьте, что файл создан Python-скриптом и имеет формат .xlsx |
| «Организация не выбрана» | Не указана организация для сверки | Выберите организацию перед запуском сверки |
| «Нет данных для выгрузки» | Попытка выгрузки до выполнения сверки | Сначала выполните сверку, затем выгружайте |

## 📞 ИНТЕГРАЦИЯ С PYTHON-СКРИПТОМ

### Полный цикл работы:

1. **Python-скрипт** (`egrn_parser.py`):
   ```bash
   python egrn_parser.py "C:\Выписки_ЕГРН" result.xlsx errors.xlsx
   ```
   → Создает файл `result.xlsx`

2. **Обработка 1С** (`СверкаЗемельныхУчастковЕГРН.epf`):
   - Загружает `result.xlsx`
   - Выполняет сверку с регистром
   - Выгружает отчет `reconciliation_report.xlsx`

3. **Бухгалтер** получает отчет с подсветкой расхождений
