## ⚙️ НАСТРОЙКИ HAProxy

### Файл: `/etc/haproxy/haproxy.cfg` (на 10.0.1.50)

```haproxy
# Админ-трафик → MASTER
acl is_admin_panel path_beg /bitrix/admin/
acl has_admin_cookie cookie BITRIX_SM_ADMIN_ID exist
acl is_admin_subdomain hdr_beg(host) -i admin.

use_backend bitrix-master if is_admin_panel
use_backend bitrix-master if has_admin_cookie
use_backend bitrix-master if is_admin_subdomain

backend bitrix-master
    server master 10.0.1.220:80 check
```

## 🛠️ ВНЕСЕНИЕ ИЗМЕНЕНИЙ В КОНФИГУРАЦИЮ

### ⚠️ ВАЖНО: Все изменения вносятся ТОЛЬКО на MASTER!

#### Порядок действий:

1. **Подключение к MASTER:**
   ```bash
   ssh root@10.0.1.220
   ```

2. **Внесение изменений:**
   ```bash
   vi /etc/php-fpm.d/www.conf
   ```

3. **Сохранение конфигурации:**
   ```bash
   /usr/local/bin/bitrix24-config.sh save "Описание изменений"
   ```

4. **Применение ко всем узлам:**
   ```bash
   /usr/local/bin/bitrix24-config.sh deploy
   ```

5. **Проверка:**
   ```bash
   /usr/local/bin/bitrix24-config.sh diff
   /usr/local/bin/bitrix24-config.sh status
   ```

## 🆘 troubleshooting

### Проблема: Админ-панель не открывается
```bash
# Проверка HAProxy
ssh root@10.0.1.50 'systemctl status haproxy'

# Проверка MASTER
ssh root@10.0.1.220 'systemctl status httpd php-fpm'

# Проверка логов
tail -50 /var/log/haproxy.log
```

### Проблема: Изменения не применяются
```bash
# Проверка синхронизации
/usr/local/bin/bitrix24-config.sh diff

# Ручная синхронизация
/root/bitrix24_rsync_sync.sh

# Проверка MD5
for ip in 10.0.1.220 10.0.1.221 10.0.1.222; do
    ssh root@$ip 'md5sum /etc/php-fpm.d/www.conf'
done
```

### Проблема: Попал не на тот сервер
```bash
# Проверка текущего узла
hostname
ip addr show | grep inet

# Принудительное подключение к MASTER
ssh root@10.0.1.220
```
