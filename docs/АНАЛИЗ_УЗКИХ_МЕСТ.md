## 🔍 ГДЕ ТОГДА ТОРМОЗА?

### Проблема скорее всего ВНЕ кластера:

1. **Клиентский интернет** 🏠
   - Медленное соединение у пользователей
   - Проверьте: `speedtest.net` с рабочего места

2. **Внешний прокси (10.0.1.110)** 🌐
   - Nginx Proxy Manager в Docker
   - Проверьте загрузку контейнера NPM

3. **Маршрут до AD сервера (10.1.20.21)** 🔐
   - LDAP авторизация может тормозить
   - Проверьте маршрут: `mtr 10.1.20.21`

4. **Браузер клиента** 🖥️
   - Кэш, расширения, слабый ПК
   - Проверьте в режиме инкогнито

5. **Внешняя сеть между офисами** 🏢
   - Если пользователи в разных локациях
   - Проверьте WAN каналы

## 🛠️ РЕКОМЕНДАЦИИ

### 1. Проверить клиентскую сеть:
```bash
# С рабочего места пользователя:
ping b24.ahprostory.ru
tracert b24.ahprostory.ru  # Windows
traceroute b24.ahprostory.ru  # Linux
speedtest.net
```

### 2. Проверить NPM (10.0.1.110):
```bash
ssh vardo001@10.0.1.110
sudo docker stats npm
```

### 3. Очистить кэш Битрикс (профилактика):
```bash
for ip in 10.0.1.220 10.0.1.221 10.0.1.222; do
  ssh root@$ip "rm -rf /home/bitrix/www/bitrix/cache/*
                 rm -rf /home/bitrix/www/bitrix/managed_cache/*"
done
```

### 4. Проверить LDAP маршрут:
```bash
ssh root@10.0.1.220
mtr -c 10 10.1.20.21  # если mtr установлен
# или
traceroute -n 10.1.20.21
```
