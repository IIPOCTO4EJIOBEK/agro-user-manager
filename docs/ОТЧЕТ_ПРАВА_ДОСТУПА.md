### Файл: `/home/bitrix/www/bitrix/php_interface/dbconn.php`

```php
// Права доступа для Unix/CGI режимов
// 0664 для файлов, 0775 для директорий - универсальные права
define("BX_FILE_PERMISSIONS", 0644);
define("BX_DIR_PERMISSIONS", 0755);
@umask(~(BX_FILE_PERMISSIONS | BX_DIR_PERMISSIONS) & 0777);
```

**Значения:**
- `BX_FILE_PERMISSIONS = 0644` (rw-r--r--)
- `BX_DIR_PERMISSIONS = 0755` (rwxr-xr-x)

## ⚠️ ЗАМЕЧАНИЯ SITE_CHECKER

### 1. check_session_ua - Warning
**Статус:** Не критично

**Причина:** Тест проверяет работу сессий без User-Agent, но:
- Сессии настроены через Redis (10.0.1.210)
- В `.settings.php` установлено `'check_ua' => false`
- Это корректная настройка для кластера

**Решение:** Не требуется. Сессии работают корректно.

### 2. check_exec - Warning
**Статус:** Не критично

**Причина:** Файл `/bitrix/site_check_exec.php` создается временно при проверке

**Решение:** Не требуется.

### 3. check_fast_download - Warning
**Статус:** Требуется включение опции в Bitrix24

**Причина:** Опция "Быстрая отдача файлов через Nginx" не включена в настройках главного модуля

**Конфигурация Nginx присутствует:**
- `/etc/nginx/bx/conf/bitrix_accel.conf` - настроен X-Accel-Redirect
- `proxy_set_header X-Sendfile-Type X-Accel-Redirect` - добавлен в заголовки

**Решение:**
1. Административная панель → Настройки → Настройки продукта → Настройки модулей → Главный модуль
2. Включить опцию "Поддержка быстрой отдачи файлов"
3. Файл `site_check_exec.php` создастся автоматически
