# СИСТЕМА СИНХРОНИЗАЦИИ BITRIX24 CLUSTER
## Дата: 24 февраля 2026

---

## 📋 ОБЗОР

Для синхронизации конфигурации между веб-узлами кластера (10.0.1.220, 221, 222) используются **три метода**:

1. **Rsync** - автоматическая синхронизация файлов
2. **Ansible** - централизованное управление конфигурацией
3. **Ручные скрипты** - для разовых операций

---

## 🔄 МЕТОOD 1: RSYNC (АВТОМАТИЧЕСКАЯ СИНХРОНИЗАЦИЯ)

### Архитектура:
```
Главный сервер (Master): 10.0.1.220
       ↓
    Rsync через SSH
       ↓
Подчиненные серверы (Slaves):
  - 10.0.1.221
  - 10.0.1.222
```

### Скрипты:

#### 1. `/usr/local/bin/bitrix24_rsync_sync.sh` - Полная синхронизация
```bash
# Запуск вручную
/usr/local/bin/bitrix24_rsync_sync.sh
```

**Синхронизируемые файлы:**
- `/etc/php-fpm.d/www.conf`
- `/etc/httpd/conf.d/php.conf`
- `/home/bitrix/www/bitrix/license_key.php`
- `/home/bitrix/www/bitrix/.settings.php`
- `/home/bitrix/www/bitrix/.settings_extra.php`
- `/home/bitrix/www/bitrix/php_interface/dbconn.php`
- `/etc/httpd/conf.d/z_bx_custom_mpm.conf`
- `/etc/httpd/conf.d/z_unique_id.conf`

#### 2. `/usr/local/bin/bitrix24_rsync_cron.sh` - Быстрая синхронизация
Запускается каждые 5 минут через cron.

### Cron задачи:
Файл: `/etc/cron.d/bitrix24_sync`
```cron
# Каждые 5 минут - быстрая синхронизация
*/5 * * * * root /usr/local/bin/bitrix24_rsync_cron.sh

# Каждый час - полная синхронизация
0 * * * * root /usr/local/bin/bitrix24_rsync_sync.sh
```

### Логирование:
- `/var/log/bitrix24_rsync_sync.log` - логи быстрой синхронизации
- `/var/log/bitrix24_rsync_full.log` - логи полной синхронизации

## 🎯 МЕТОД 2: ANSIBLE (ЦЕНТРАЛИЗОВАННОЕ УПРАВЛЕНИЕ)

### Запуск playbook:
```bash
cd /root
ansible-playbook -i /etc/ansible/hosts bitrix24_sync.yml
```

### Файлы:
- `/root/bitrix24_sync.yml` - playbook для синхронизации
- `/etc/ansible/hosts` - инвентарь кластера

### Преимущества:
- ✅ Идемпотентность (повторяемость)
- ✅ Проверка состояния перед применением
- ✅ Откат изменений при ошибках
- ✅ Централизованное управление

### Группы хостов:
```ini
[bitrix-web]
b24-web-01 ansible_ssh_host=10.0.1.220
b24-web-02 ansible_ssh_host=10.0.1.221
b24-web-03 ansible_ssh_host=10.0.1.222
```

## 🚀 ИСПОЛЬЗОВАНИЕ

```bash
# Настроена через cron
```

```bash
# Быстрая (только критичные файлы)
/usr/local/bin/bitrix24_rsync_sync.sh

# Полная (все файлы)
/usr/local/bin/bitrix24_rsync_sync.sh

```

```bash

ssh root@10.0.1.222 'systemctl status php-fpm httpd'
```

## ⚠️ ВОЗМОЖНЫЕ ПРОБЛЕМЫ

### 1. Ошибка SSH
**Симптомы:** Не подключается к серверам
**Решение:** Проверить доступность и пароль

### 2. Конфликт версий
**Симптомы:** Разные версии PHP на узлах
**Решение:** `dnf update php-fpm` на всех узлах

### 3. Блокировка файлов
**Симптомы:** Файлы не синхронизируются
**Решение:** Проверить права доступа и процессы
