## 🔍 ПОЧЕМУ МОЖЕТ ТОРМОЗИТЬ?

### Возможные причины:

1. **Внешний прокси (10.0.1.110)**
   - Nginx Proxy Manager может быть узким местом
   - Проверьте загрузку CPU/RAM на 10.0.1.110

2. **Клиентский интернет**
   - Медленное соединение у пользователей

3. **LDAP авторизация**
   - Таймауты при подключении к AD (10.1.20.21)

4. **Битрикс кэш**
   - Требуется очистка

5. **HAProxy логи**
   - Проверьте: tail -100 /var/log/haproxy.log

## 🛠️ РЕКОМЕНДАЦИИ

### 1. Очистить кэш Битрикс:
```bash
for ip in 10.0.1.220 10.0.1.221 10.0.1.222; do
  ssh root@$ip "rm -rf /home/bitrix/www/bitrix/cache/*
                 rm -rf /home/bitrix/www/bitrix/managed_cache/*
                 rm -rf /home/bitrix/www/bitrix/stack_cache/*"
done
```

### 2. Проверить внешний прокси 10.0.1.110:
```bash
ssh root@10.0.1.110
top -bn1 | head -20
```

### 3. Проверить логи на ошибки:
```bash
# Apache
tail -200 /var/log/httpd/error_log | grep -iE 'error|warn|timeout'

# HAProxy
tail -100 /var/log/haproxy.log
```

### 4. Оптимизировать PHP-FPM (если нужно):
```bash
# Увеличить количество процессов при высокой нагрузке
pm.max_children = 200  # вместо 100
pm.start_servers = 40  # вместо 20
```
