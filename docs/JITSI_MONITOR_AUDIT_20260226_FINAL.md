## ⚠️ ОБНАРУЖЕННЫЕ ПРОБЛЕМЫ

### 1. 🔴 Узел 10.0.1.132 НЕДОСТУПЕН

**Симптомы:**
```
ssh: connect to host 10.0.1.132 port 22: No route to host
ping: 100% packet loss
Destination Host Unreachable
```

**Влияние:**
- Мониторинг показывает только JVB Node 131
- При отказе 132 вся нагрузка на 131
- Возможна потеря конференций на 132

**Требуется:**
```bash
# Проверить физическую доступность
ssh vardo001@10.0.1.1 "ip route get 10.0.1.132"

# Проверить фаервол на 132
# Проверить статус сервера (возможно выключен)
```

### 2. 🟡 Лог Prosody отсутствует

**Проблема:**
```
grep: /config/prosody.log: No such file or directory
```

**Влияние:**
- Не собираются Display Names участников
- В интерфейсе показываются ID вместо имен

**Решение (обновить app.py):**
```python
# Использовать Docker логи вместо файла
cmd = "docker logs jitsi-prosody-1 2>&1 | grep nick"
```

### 3. 🟡 GeoIP API может быть недоступен

**Проблема:**
```python
r = requests.get(f'http://ip-api.com/json/{ip}?fields=countryCode,city', timeout=0.5)
```

**Влияние:**
- При недоступности API показывается "Unknown"
- Таймаут 0.5с может быть мал для медленных сетей
