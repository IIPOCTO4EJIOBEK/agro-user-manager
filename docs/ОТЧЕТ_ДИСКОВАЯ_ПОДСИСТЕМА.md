## 🛠️ РЕКОМЕНДАЦИИ

### 1. Проверить агенты Bitrix:
```sql
-- Подключиться к БД на 10.0.1.200
mysql -u root prostory

-- Посмотреть активные агенты
SELECT ID, MODULE_ID, AGENT_INTERVAL, NEXT_EXEC 
FROM b_agent 
WHERE ACTIVE='Y' 
ORDER BY NEXT_EXEC 
LIMIT 20;

-- Посмотреть количество событий
SELECT COUNT(*) FROM b_event;
```

### 2. Оптимизировать cron:
```bash
# Проверить частоту запуска
cat /etc/cron.d/bitrix*

# Если запуск каждую минуту - нормально
* * * * * bitrix /usr/bin/php -f /home/bitrix/www/bitrix/modules/main/tools/cron_events.php
```

### 3. Расширить диск (если нужно):
```bash
# На хосте VM добавить диск
# Затем на гостевой:
pvcreate /dev/sdb
vgextend rlm /dev/sdb
lvextend -L +50G /dev/rlm/root
resize2fs /dev/rlm/root
```

### 4. Разделить кэш (опционально):
```php
// В .settings.php
'cache' => [
  'value' => [
    'type' => 'memcached',  // вместо файлов
    'servers' => [
      '10.0.1.210:11211'
    ]
  ]
]
```
