# ⚠️ ВАЖНО: УПРАВЛЕНИЕ КОНФИГУРАЦИЕЙ BITRIX24 CLUSTER

## ❌ ПРОБЛЕМА

**Если вы измените файл напрямую на 10.0.1.221 или 10.0.1.222, ваше изменение будет ПОТЕРЕРЯНО при следующей автоматической синхронизации!**

### Почему:
```
Автоматическая синхронизация (каждые 5 мин):
10.0.1.220 (MASTER) → 10.0.1.221, 10.0.1.222 (SLAVE)

Ваши изменения на 221/222 будут перезаписаны файлом с 220!
```

---

## ✅ ПРАВИЛЬНЫЙ ПОРЯДОК ВНЕСЕНИЯ ИЗМЕНЕНИЙ

### Используйте скрипт `bitrix24-config.sh`

**Расположение:** `/usr/local/bin/bitrix24-config.sh` (на 10.0.1.220)

### Команды:

#### 1. Проверка различий
```bash
# На 10.0.1.220
/usr/local/bin/bitrix24-config.sh diff
```

#### 2. Внесение изменений на MASTER
```bash
# Подключитесь к 10.0.1.220
ssh root@10.0.1.220

# Измените файл
vi /etc/php-fpm.d/www.conf

# Сохраните конфигурацию
/usr/local/bin/bitrix24-config.sh save "Увеличено pm.max_children до 150"
```

#### 3. Применение изменений
```bash
# На 10.0.1.220
/usr/local/bin/bitrix24-config.sh deploy
```

#### 4. Проверка статуса
```bash
/usr/local/bin/bitrix24-config.sh status
```

## 📋 ПРИМЕР ПОЛНОГО ЦИКЛА

```bash
# 1. Подключение к MASTER
ssh root@10.0.1.220

# 2. Проверка текущей конфигурации
/usr/local/bin/bitrix24-config.sh diff

# 3. Внесение изменений
vi /etc/php-fpm.d/www.conf
# Измените pm.max_children = 100 на pm.max_children = 150

# 4. Сохранение конфигурации
/usr/local/bin/bitrix24-config.sh save "Увеличено pm.max_children до 150"

# 5. Применение изменений ко всем узлам
/usr/local/bin/bitrix24-config.sh deploy

# 6. Проверка
/usr/local/bin/bitrix24-config.sh status
/usr/local/bin/bitrix24-config.sh diff
```

## 🆘 ЭКСТРЕННАЯ ПОМОЩЬ

### Если изменения потеряны:
1. Проверьте логи: `tail -50 /var/log/bitrix24_rsync_sync.log`
2. Восстановите из резервной копии: `ls /etc/php-fpm.d/www.conf.bak.*`
3. Примените конфигурацию заново: `bitrix24-config.sh deploy`

### Если узел не синхронизируется:
1. Проверьте доступность: `ping 10.0.1.221`
2. Проверьте службы: `ssh root@10.0.1.221 'systemctl status php-fpm httpd'`
3. Запустите вручную: `/root/bitrix24_rsync_sync.sh`
