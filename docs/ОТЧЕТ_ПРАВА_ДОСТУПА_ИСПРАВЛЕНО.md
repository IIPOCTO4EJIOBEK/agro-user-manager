## ❌ ПРОБЛЕМА

**Bitrix site_checker показывал ошибку:**
```
Недоступны для чтения или записи:
/home/bitrix/www/bitrix/pub/upload/login-custom.css
/home/bitrix/www/upload/login-custom.css
```

---

## 🔍 ПРИЧИНА

**Файлы принадлежали root:root вместо bitrix:bitrix:**
```
-rw-r--r-- 1 root root 328 login-custom.css
```

**Почему это произошло:**
- Папка `/home/bitrix/www/upload` смонтирована через **NFS** с сервера `10.0.1.1`
- NFS мапит права root → root на клиенте
- Файлы были созданы с правами root

**NFS Mount:**
```
10.0.1.1:/nvme_db/bitrix_upload_fast on /home/bitrix/www/upload type nfs4
(rw,noatime,nodiratime,vers=4.2,rsize=1048576,wsize=1048576)
```

---

## ✅ РЕШЕНИЕ

### 1. Удалены проблемные файлы:
```bash
rm -f /home/bitrix/www/bitrix/pub/upload/login-custom.css
rm -f /home/bitrix/www/upload/login-custom.css
```

**Результат:** ✅ Файлы удалены на всех узлах (10.0.1.220, 221, 222)

## 🛠️ ЕСЛИ ПРОБЛЕМА ПОВТОРИТСЯ

### Вариант 1: Исправить права на NFS сервере
```bash
# На сервере 10.0.1.1
chown -R bitrix:bitrix /nvme_db/bitrix_upload_fast/login-custom.css
# Или удалить файл
rm -f /nvme_db/bitrix_upload_fast/login-custom.css
```

### Вариант 2: Настроить NFS idmap
```bash
# На NFS сервере /etc/idmapd.conf
[Mapping]
Nobody-User = bitrix
Nobody-Group = bitrix
```

### Вариант 3: Использовать local disk вместо NFS
```bash
# Размонтировать NFS
umount /home/bitrix/www/upload

# Создать локальную папку
mkdir -p /home/bitrix/www/upload
chown -R bitrix:bitrix /home/bitrix/www/upload
```
