# Инструкция по запуску парсера выписок ЕГРН

## Требования

Для работы скрипта необходимы следующие Python-библиотеки:

```bash
pip install pdfplumber pandas openpyxl
```

## Быстрый старт

### Вариант 1: Запуск с настройками по умолчанию

Скрипт будет искать PDF-файлы в папке `C:\Выписки_ЕГРН` и сохранит результат в текущей директории:

```bash
python egrn_parser.py
```

### Вариант 2: Запуск с указанием параметров

```bash
python egrn_parser.py "Путь/к/папке/с/PDF" "output.xlsx" "errors.xlsx"
```

Пример для Linux/Mac:
```bash
python egrn_parser.py "/home/user/egrn_pdfs" "result.xlsx" "parse_errors.xlsx"
```

Пример для Windows:
```bash
python egrn_parser.py "C:\Мои_выписки\ЕГРН" "отчет.xlsx" "ошибки.xlsx"
```

## Выходные файлы

После успешного выполнения скрипт создаст:

1. **egrn_parsed_data.xlsx** (или указанное имя) — основной файл с извлеченными данными:
   - Файл_источник — имя исходного PDF файла
   - №_участка_в_файле — порядковый номер участка в файле (если их несколько)
   - Кадастровый_номер — кадастровый номер земельного участка
   - ОКТМО — код муниципального образования
   - Категория_земель — текстовое описание категории
   - Код_категории_земель — цифровой код категории (если найден)
   - Кадастровая_стоимость — стоимость в рублях
   - Доля — доля в праве собственности (например, 1/1)
   - Дата — дата выписки или регистрации права

2. **egrn_parse_errors.xlsx** (или указанное имя) — отчет об ошибках парсинга:
   - Файл — имя файла, где возникла проблема
   - Секция — номер секции в файле
   -Ошибка — описание проблемы
   - Дата_время — когда произошла ошибка

3. **egrn_parser.log** — лог-файл с подробной информацией о процессе выполнения

## Структура данных для 1С

Экспортируемый Excel-файл готов к загрузке в 1С. Рекомендуемые типы данных для обработки 1С:

| Поле Excel | Тип 1С | Примечание |
|------------|--------|------------|
| Кадастровый_номер | Строка(50) | Ключевое поле для поиска |
| ОКТМО | Строка(11) | Без пробелов |
| Категория_земель | Справочник.КатегорииЗемель | Текстовое соответствие |
| Код_категории_земель | Строка(12) | Для автоматической проверки ставки |
| Кадастровая_стоимость | Число(15,2) | В рублях |
| Доля | Строка(20) | Формат "1/1", "1/2" и т.д. |
| Дата | Дата | Формат ДД.ММ.ГГГГ |

## Проверка налоговой ставки

В скрипте определены коды категорий земель для сельхозназначения (ставка 0.3%):

- `003001000000` — Земли сельскохозяйственного назначения
- `003001001000` — Сельскохозяйственные угодья  
- `003001002000` — Земли, занятые объектами сельскохозяйственного назначения
- `003001003000` — Пашни, сенокосы, пастбища, залежи

Для всех остальных категорий применяется ставка 1.5%.

Эту логику необходимо воспроизвести в обработке 1С.

## Рекомендации по использованию

1. **Подготовка данных**: Скачайте все выписки из личного кабинета Росреестра в одну папку.

2. **Проверка формата**: Убедитесь, что выписки в формате PDF (не сканы, а машиночитаемые документы).

3. **XML вместо PDF**: Если есть возможность выгрузки в XML через личный кабинет Росреестра — используйте её. XML-парсинг будет значительно надежнее.

4. **Тестовый запуск**: Сначала протестируйте на небольшой выборке (5-10 файлов), проверьте корректность извлечения.

5. **Ручная проверка**: После первого запуска выборочно проверьте 3-5 участков сверкой с оригинальными выписками.

## Возможные проблемы и решения

### Проблема: Не извлекается кадастровый номер
**Решение**: Проверьте, является ли PDF машиночитаемым (можно ли выделить текст). Если это скан — потребуется OCR.

### Проблема: Неправильно определяется категория земель
**Решение**: Откройте файл лога, найдите проблемный файл, проверьте как записана категория в оригинале. При необходимости добавьте новый regex-шаблон.

### Проблема: Ошибка при чтении PDF
**Решение**: Файл может быть поврежден или защищен паролем. Проверьте файл вручную.

### Проблема: Пустой результат
**Решение**: 
- Проверьте путь к папке с выписками
- Убедитесь, что файлы имеют расширение `.pdf` или `.PDF`
- Проверьте лог-файл на наличие ошибок

## Масштабирование

Для больших объемов (сотни/тысячи участков):
- Скрипт обрабатывает файлы последовательно, время выполнения ~1-2 секунды на файл
- Рекомендуется разбивать на пакеты по 100-200 файлов
- Логи сохраняются, можно возобновить с места остановки

## Контакты для доработки

При изменении формата выписок ЕГРН может потребоваться обновление regex-шаблонов в методах класса `EGRNParser`.
