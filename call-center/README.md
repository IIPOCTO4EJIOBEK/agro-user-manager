# Call Center System

Система управления колл-центром с поддержкой до 7 операторов, приоритезацией звонков и подробной аналитикой.

## Возможности

### Основные функции
- ✅ Поддержка до 7 операторов в смену
- ✅ Индивидуальная настройка графика для каждого оператора
- ✅ Персонификация операторов (отдельные профили, статистика)
- ✅ Три уровня приоритета звонков:
  - **Приоритет 1 (VIP)** - эскалация на руководителя через 30 сек
  - **Приоритет 2 (Важные)** - уборочная, бухгалтеры
  - **Приоритет 3 (Обычные)** - все остальные
- ✅ Очередь звонков с приоритезацией (FIFO внутри приоритета)
- ✅ Маршрутизация Round Robin
- ✅ Переадресация на других операторов и мобильные номера
- ✅ Статусы операторов: Свободен, Занят, Обработка заявки, Перерыв, Оффлайн
- ✅ Уведомление о звонках длительностью более 5 минут

### Отчетность и аналитика
- 📊 Количество звонков на оператора за период
- 📊 Средняя длительность звонка
- 📊 Время ожидания в очереди
- 📊 Количество переадресованных звонков
- 📊 Время перерывов
- 📊 Экспорт в CSV/JSON
- 📊 Интеграция через API

## Установка

```bash
cd backend
npm install
```

## Запуск

### Backend сервер
```bash
cd backend
npm start
```

Сервер запустится на порту 3000 (REST API) и 8080 (WebSocket).

### Frontend интерфейсы

Откройте в браузере:

1. **Панель оператора**: `frontend/operator.html`
2. **Панель администратора**: `frontend/admin.html`  
3. **Панель руководителя**: `frontend/manager.html`

Для работы frontend необходимо открыть файлы через локальный веб-сервер или использовать Live Server.

## API Endpoints

### REST API
- `GET /api/operators` - список операторов
- `POST /api/operators` - добавить оператора
- `GET /api/calls` - список звонков
- `GET /api/queue` - очередь звонков
- `GET /api/analytics?period=day|week|month` - аналитика
- `GET /api/analytics/export?format=csv` - экспорт данных
- `POST /api/calls/incoming` - симуляция входящего звонка

### WebSocket (порт 8080)

Сообщения клиент → сервер:
```json
{ "type": "login", "operatorId": "op1" }
{ "type": "logout", "operatorId": "op1" }
{ "type": "setStatus", "operatorId": "op1", "status": "free|busy|break|processing" }
{ "type": "answerCall", "operatorId": "op1", "callId": "..." }
{ "type": "transferCall", "callId": "...", "targetPhone": "+7..." }
{ "type": "endCall", "operatorId": "op1", "callId": "..." }
{ "type": "incomingCall", "callerNumber": "+7...", "priority": 1|2|3 }
```

## Сценарии использования

### Выход на смену
1. Оператор открывает `operator.html`
2. Выбирает свой профиль из списка
3. Нажимает "Войти" - статус автоматически становится "Свободен"
4. Система начинает распределять звонки

### Обработка вызова
1. Поступает звонок → добавляется в очередь с учетом приоритета
2. Оператор со статусом "Свободен" видит звонок в очереди
3. Оператор нажимает "Ответить"
4. После завершения - статус автоматически "Обработка заявки"
5. Оператор вручную ставит "Свободен"

### VIP звонок
1. Звонок приоритета 1 отображается всем операторам
2. Любой свободный оператор может ответить
3. Если никто не ответил за 30 сек → эскалация на менеджера

### Перерыв
1. Оператор нажимает "Перерыв"
2. Система прекращает распределение звонков
3. По возвращении - статус "Свободен"

## Тестирование

### Симуляция входящего звонка
```bash
curl -X POST http://localhost:3000/api/calls/incoming \
  -H "Content-Type: application/json" \
  -d '{"callerNumber": "+79991234567", "priority": 1}'
```

### Проверка аналитики
```bash
curl http://localhost:3000/api/analytics?period=day
```

## Масштабирование

Для подключения реального телефона 8-800:
1. Настройте SIP-транк от провайдера (Ростелеком)
2. Интегрируйте с Asterisk/FreePBX или облачной АТС
3. Используйте webhook для передачи событий в систему

## Опциональная интеграция

### SD Plus
- Автоматическое создание заявок при звонке
- Привязка оператора и инициатора

### Системы отчетности
- REST API для выгрузки данных
- Периодическая синхронизация по расписанию

## Требования к ресурсам

- **Оперативная память**: ~50MB для backend
- **Процессор**: 1 ядро достаточно для 7 операторов
- **Хранилище**: ~10MB для данных
- **Сеть**: минимальные требования, WebSocket для real-time

- Реализовать ролевую модель доступа
